模块化

模块化

  1. 到目前为止，对于编写源码来说，主流的方案只剩下三种。

    esm: 从 ES6 起官方规范自带的方案
cjs: Node.js 使用的方案
IIFE: 一个自动执行的功能，适合作为<script>标签。（对于导出包来说，依赖视为已经准备好，直接从 window/global 取值并传入。）

  2. Tree-shaking的本质是消除无用的js代码 在 webpack 项目中，有一个入口文件，相当于一棵树的主干，入口文件有很多依赖的模块，相当于树枝。实际情况中，虽然依赖了某个模块，但其实只使用其中的某些功能。通过 tree-shaking，将没有使用的模块摇掉，这样来达到删除无用代码的目的。
